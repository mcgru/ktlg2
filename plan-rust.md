# План каталогизации

## Что было раньше

Анализ `old.bash/ktlg2.sh` — bash-скрипт для каталогизации медиафайлов.

### Режимы работы

| Флаг | Назначение |
|---|---|
| `-t` / `--time` | Выставить `touch` в timestamp из метаданных для всех файлов |
| `-c` / `--check` | Проверить соответствие имён файлов их датам |
| `--check-dates` | Вывести список файлов с датами |
| `-r` / `--rename` | Переименовать в формат `YYMMDD-HHMMSS.ХХХХХХХ.z.ext` |
| `-p` / `--plane` | Собрать все файлы из поддиректорий в корень |
| `-d` / `--dup` | Найти дубликаты через `fdupes` |
| `--force` | Пропускать ошибки при проверке |

### Приоритет источников даты

1. **EXIF / ffprobe** — метаданные изображений (exif) и видео (ffprobe)
2. **Имя файла** — парсинг даты из имени (YYYYMMDD, YYMMDD и т.д.)
3. **Файловая система** — `stat` mtime

### Логика раскладки по папкам (основной режим)

1. Создаётся директория `$PATH.years`
2. Для каждого файла определяется год (`YYYY`) и месяц (`YYYY-MM`)
3. Из пути извлекается суффикс (очистка числовых префиксов в именах папок)
4. Файл перемещается в `$PATH.years/$Год.суффикс/$Год-Месяц/`
5. **Коллизии**: если файл с таким именем уже есть — `diff` проверяет содержимое. Дубликат → удаляется, разный → старый переименовывается с суффиксом `.(n).`
6. Пустые директории удаляются
7. Финальное переименование: `$PATH.years` → `$Годы.суффикс` (или `$Год-min-Год-max.суффикс` при нескольких годах)

### Пример

```
исходное: ./photos/ (img_1234.jpg 2023-05-10, video.MP4 2022-08-15)
результат: ./photos/ → ./photos.problems/<timestamp>
           ./2022-2023/2022/2022-08/video.MP4
           ./2022-2023/2023/2023-05/img_1234.jpg
```

### Зависимости

`exif`, `ffprobe` (ffmpeg), `identify` (ImageMagick), `fdupes`, `parallel` (GNU parallel).

## Что нужно сделать

Переписать `ktlg2.sh` на Rust с сохранением всей функциональности.

### CLI-дизайн

Одна команда, подкоманды для режимов:

```
ktlg2 [OPTIONS] <PATH>
ktlg2 organize [OPTIONS] <PATH>   # разложить по папкам год/месяц (режим по умолчанию)
ktlg2 rename  [OPTIONS] <PATH>   # переименовать файлы
ktlg2 check   [OPTIONS] <PATH>   # проверить соответствие имён и дат
ktlg2 plane   [OPTIONS] <PATH>   # собрать файлы из поддиректорий в корень
ktlg2 dups    [OPTIONS] <PATH>   # найти дубликаты
ktlg2 touch   [OPTIONS] <PATH>   # выставить file mtime из метаданных

Общие опции:
  -j, --jobs <N>    количество потоков (по умолчанию: число CPU)
  --dry-run         не перемещать/переименовывать, только показать
  -v, --verbose     подробный вывод
  --json            вывод в JSON (для dups и check)
```

### Архитектура

```
src/
  main.rs      — точка входа, разбор аргументов (clap), диспетчеризация
  extract.rs   — извлечение timestamp из файла
  organize.rs  — раскладка по папкам, коллизии
  rename.rs    — переименование в канонический формат
  dups.rs      — поиск дубликатов (сравнение хэшей)
```

Каждый файл — один модуль с одной ответственностью. Кода в каждом — до 200 строк.

### Извлечение даты (`extract.rs`)

Приоритет (как в оригинале):

1. **EXIF** (изображения: jpg, jpeg, tiff, png) — `kamadak-exif`, поле `DateTimeOriginal`
2. **Контейнер MP4** (mp4, mov) — `mp4parse`, поле `creation_time` из `mvhd`
3. **Имя файла** — регулярки: `YYYYMMDD`, `YYMMDD-HHMMSS` и т.д.
4. **Файловая система** — `std::fs::Metadata::modified()` (mtime)

Для AVI и других форматов без библиотечной поддержки — пропускаем 1-2, идём сразу к имени файла и ФС.

```rust
pub struct FileTimestamp {
    pub timestamp: i64,       // unix seconds
    pub source: TimestampSource,
}

pub enum TimestampSource {
    Exif,
    Mp4Container,
    Filename,
    Filesystem,
}

pub fn extract_timestamp(path: &Path) -> Option<FileTimestamp>
```

### Параллелизация

Rayon — `.par_iter()` вместо `.iter()`.

Управление числом потоков через `ThreadPoolBuilder::new().num_threads(N)` при старте (параметр `--jobs`).

### Раскладка по папкам (`organize.rs`)

Алгоритм (повторяет bash-оригинал):

1. Собрать все файлы рекурсивно (`walkdir`)
2. Для каждого извлечь timestamp
3. Извлечь суффикс из пути: взять относительный путь, убрать числовые префиксы из имён папок, склеить через `.`
4. Целевая папка: `<base>.years/<год>.<суффикс>/<год-месяц>/
5. При коллизии имён:
   - Если содержимое совпадает (сравнить MD5) — удалить дубликат
   - Если разное — переименовать существующий файл с суффиксом `.1`, `.2`...
6. Финальное переименование: `<base>.years` → `<год[-год_конечный]>.суффикс`

### Переименование (`rename.rs`)

Формат: `YYMMDD-HHMMSS.ХХХХХХХ.z.ext`

Где `ХХХХХХХ` — первые 7 символов MD5 содержимого файла (как в оригинале — `dd if= count=50 | md5sum | cut -c1-7`).

### Поиск дубликатов (`dups.rs`)

- Полный MD5 файла (не обрезанный, в отличие от rename)
- Группировка по размеру → по хэшу → вывод групп
